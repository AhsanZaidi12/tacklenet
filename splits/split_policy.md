# TackleNet — Official Split Policy

**Version:** 1.0 (Frozen)  
**Applies to:** The official benchmark split for the current public TackleNet release.

---

## Purpose

This document defines the **official train / validation / test partition** for TackleNet.
All benchmark results intended to be comparable across methods, papers, and code releases should use:

```text
splits_frozen.csv
```

For official reporting, users should **not regenerate the split** and should **not substitute a custom partition**.
Alternative splits may be used for exploratory analysis, but such results should be clearly labeled as **non-official** and should not be reported as directly comparable to the released benchmark.

---

## Authoritative split file

The authoritative partition assignment for each clip is stored in:

```text
splits_frozen.csv
```

This file assigns every released clip to exactly one of:
- `train`
- `val`
- `test`

It is the single source of truth for benchmark evaluation in the released dataset.

---

## Split method

| Property | Value |
|---|---|
| Method | Stratified clip-level split |
| Stratification label | `satt3_risk_binary` |
| Ratios | 70% train / 15% val / 15% test |
| Random seed | `42` |
| Total clips | 737 |
| Generation script | `tacklenet_create_splits_v2.py` |

The split was generated by stratifying on the official binary classification label `satt3_risk_binary` so that class proportions remain closely matched across train, validation, and test partitions.

These counts define the frozen benchmark split and should be preserved in all official uses of the dataset.

## Why the split is clip-level rather than session-level

A grouped split by practice session would be preferable because it would reduce the risk of leakage across clips that share session context, camera setup, drill conditions, or participants.

However, reliable session identifiers were **not available in the released annotation file at the time of the dataset freeze**. As a result, a grouped session-level split could not be constructed reproducibly from the public release. The official split therefore uses **clip-level stratified sampling**.

This limitation should be acknowledged in downstream work using the benchmark.

### Recommended limitation statement

> Because reliable session identifiers were not available in the released annotation file at the time of the dataset freeze, the official TackleNet split is stratified at the clip level rather than grouped by session. Clips from the same practice session may therefore appear in different partitions. This is a known limitation of the current public release.

---

## Benchmark usage rules

- Use **only** the clips assigned to `train` for model fitting.
- Use **only** the clips assigned to `val` for model selection, threshold tuning, ablations, and early stopping.
- Use **only** the clips assigned to `test` for final held-out reporting.
- Do not move clips across partitions.
- Do not tune on the test split.
- For benchmark comparability, all Tasks A, B, and C should use the same frozen partition file.

## Reproducibility

To verify the frozen split from the released annotation file, run:

```bash
python tacklenet_create_splits_v2.py \
    --input annotations_satt_fpoc.csv \
    --output splits_check.csv \
    --seed 42
```

When run on the same annotation file with the same script version and seed, the generated assignment should match `splits_frozen.csv`.

---

## Integrity expectations

The released frozen split satisfies the following checks:

- total rows in `splits_frozen.csv` = **737**
- every released `clip_id` appears **exactly once**
- no clip is assigned to more than one partition
- the `clip_filename` field matches the zero-padded MP4 filename for the clip
- class proportions are closely matched across train, validation, and test

Users are encouraged to verify these checks after download.

## How to load the split

```python
import pandas as pd

ann = pd.read_csv("annotations_satt_fpoc.csv")
splits = pd.read_csv("splits_frozen.csv")

data = ann.merge(
    splits[["clip_id", "split"]],
    on="clip_id",
    how="inner"
)

train_df = data[data["split"] == "train"].copy()
val_df   = data[data["split"] == "val"].copy()
test_df  = data[data["split"] == "test"].copy()
```

The `clip_filename` column in `splits_frozen.csv` provides the zero-padded video filename corresponding to each `clip_id` (for example, `042.mp4`).

## Notes for citation and reporting

Any paper, report, or benchmark submission using the official TackleNet split should:

1. state that `splits_frozen.csv` was used,
2. report that the split is **stratified clip-level**,
3. acknowledge the lack of public session identifiers as a limitation, and
4. avoid claiming session-level leakage control unless additional private metadata was used.
